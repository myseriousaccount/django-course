# Шаблони: вбудовані теги й фільтри

Шаблон — це не просто HTML: у ньому можна ухвалювати рішення (`if`), проходити по списках (`for`), підключати частини й підставляти дані. Django дає для цього готовий набір **тегів** (`{% ... %}`) і **фільтрів** (`|`). Цей урок — практичний довідник по тому, що ти використовуватимеш у кожному шаблоні. Приклади навмисно з **різних доменів** (блог, магазин, кіно, форум, проста сторінка), щоб було видно: це універсальний інструмент.

## Виведення значень: {{ }}

> **`{{ змінна }}`** — виводить значення в HTML. Django сам екранує небезпечні символи (`<`, `>`, `&`), захищаючи від XSS. Крапка `{{ post.author.username }}` іде вглиб: атрибут → ключ словника → елемент списку → виклик методу **без дужок**.

```django
<h1>{{ movie.title }}</h1>
<p>Режисер: {{ movie.director.full_name }}</p>   {# метод, але БЕЗ дужок #}
<p>У кошику: {{ cart.items.count }} товарів</p>
```

> <i class="bi bi-exclamation-triangle"></i> У Django-шаблоні **не можна викликати методи з аргументами** й немає круглих дужок: пишеш `{{ obj.get_name }}`, а не `{{ obj.get_name() }}`. Django свідомо обмежує логіку в шаблонах — складне має жити у view.

## Теги логіки: умови

> **Тег** у шаблоні — це конструкція у `{% ... %}`, що виконує логіку (розгалуження, цикл, підключення). На відміну від `{{ ... }}`, який лише **виводить** значення, тег **керує** тим, що потрапить у результат.

**Визначення.** Умовний блок показує різні шматки шаблону залежно від того, чи істинна умова.

**Як це працює.** Синтаксис близький до Python, але з тегами замість двокрапки й відступів:

```django
{% if movie.rating >= 8 %}
  <span class="badge gold">Хіт</span>
{% elif movie.rating >= 5 %}
  <span class="badge">Непогано</span>
{% else %}
  <span class="badge grey">На любителя</span>
{% endif %}
```

Усередині умов працюють оператори: `==`, `!=`, `<`, `>`, `<=`, `>=`, `and`, `or`, `not`, а також `in`:

```django
{% if user in movie.fans.all %}Ви вподобали цей фільм{% endif %}
{% if not comments %}Коментарів ще немає{% endif %}
```

**Навіщо.** Один шаблон обслуговує кілька станів даних — не треба тримати окремий HTML для кожного випадку.

> <i class="bi bi-exclamation-triangle"></i> Кожен блок логіки треба **закривати**: `{% endif %}`, `{% endfor %}`. Django не орієнтується на відступи, тож без закриття шаблон не скомпілюється.

## Теги логіки: цикли

**Визначення.** `{% for %}` проходить по колекції (список, QuerySet) і повторює вкладений блок для кожного елемента. `{% empty %}` спрацьовує, якщо колекція порожня.

**Як це працює** (список коментарів на форумі):

```django
<ul>
  {% for comment in comments %}
    <li>{{ forloop.counter }}. <b>{{ comment.author }}</b>: {{ comment.text }}</li>
  {% empty %}
    <li>Коментарів поки немає — будьте першим.</li>
  {% endfor %}
</ul>
```

Усередині циклу доступна змінна `forloop`:

| Поле | Що дає |
|---|---|
| `forloop.counter` | номер ітерації з **1** |
| `forloop.counter0` | номер ітерації з **0** |
| `forloop.revcounter` | номер із кінця |
| `forloop.first` | `True` на першій ітерації |
| `forloop.last` | `True` на останній ітерації |
| `forloop.parentloop` | доступ до зовнішнього циклу (у вкладених) |

**Навіщо.** `{% empty %}` рятує від класичної проблеми «порожня сторінка без пояснення»: замість голого місця користувач бачить зрозуміле повідомлення. А `forloop.counter` дає нумерацію без зайвої логіки у view.

> <i class="bi bi-lightbulb"></i> Уяви касира, що пробиває товари з кошика по одному. `forloop.counter` — це лічильник на його табло: скільки вже пробито. `forloop.last` — сигнал «це останній товар, час рахувати підсумок».

## Наслідування шаблонів: extends, block, include

> **Наслідування** дозволяє винести спільний каркас (шапка, меню, підвал) у `base.html`, а на дочірніх сторінках заповнювати лише «дірки» — блоки.

**Як це працює.**

```django
{# base.html — спільний каркас #}
<html>
  <body>
    <nav>...меню...</nav>
    <main>{% block content %}{% endblock %}</main>
    <footer>...</footer>
  </body>
</html>
```

```django
{# movie_list.html — дочірня сторінка #}
{% extends 'base.html' %}

{% block content %}
  <h1>Фільми</h1>
  {% for movie in movies %}{{ movie.title }}{% endfor %}
{% endblock %}
```

| Тег | Роль |
|---|---|
| `{% extends 'base.html' %}` | «вбудуватися в каркас» — **перший рядок** дочірнього шаблону |
| `{% block name %}...{% endblock %}` | «дірка», яку заповнює дочірній шаблон |
| `{% include 'partials/card.html' %}` | «вставити частину сюди» — для повторюваних блоків |

**Різниця `extends` і `include`.** `extends` — це «я стаю частиною спільного каркаса». `include` — це «встав сюди готовий шматок». Картка товару, картка фільму, елемент меню — це `include`; загальна обгортка сторінки — `extends`.

> <i class="bi bi-info-circle"></i> `include` уміє передавати дані: `{% include 'card.html' with item=movie %}`. Так одна картка обслуговує і фільм, і товар, і статтю.

## Теги посилань і статики: url, static, csrf_token

**Визначення.** Ці теги вставляють у сторінку правильні URL, статичні файли та захисний токен — без ручного прописування шляхів.

**Як це працює.**

```django
{% load static %}

<!-- Посилання за ІМЕНЕМ маршруту, а не за жорстким URL -->
<a href="{% url 'movie_detail' movie.id %}">Детальніше</a>
<a href="{% url 'article_detail' slug=post.slug %}">Читати</a>

<!-- Шлях до статичного файлу (CSS, JS, зображення) -->
<link rel="stylesheet" href="{% static 'css/style.css' %}">
<img src="{% static 'img/logo.png' %}" alt="Лого">

<!-- Захисний токен у формах -->
<form method="post">
  {% csrf_token %}
  ...
</form>
```

Розбір кожного:

- `{% url 'name' %}` — будує адресу за **іменем** маршруту з `urls.py`. Зміниш URL у `urls.py` — усі посилання оновляться самі. Аргументи передають позиційно (`movie.id`) або за іменем (`slug=post.slug`).
- `{% static '...' %}` — повертає правильний шлях до файлу зі `STATIC`. Потребує `{% load static %}` угорі шаблону.
- `{% csrf_token %}` — обов'язковий у кожній формі з `method="post"`; без нього Django відхилить запит із помилкою **403**.

**Навіщо.** Ти працюєш з **іменами й логічними шляхами**, а не з фізичними адресами. Це і є конвенція над конфігурацією: структура вже узгоджена, ти лише посилаєшся на неї.

## Інші корисні теги

| Тег | Робить |
|---|---|
| `{% with total=order.items.count %}...{% endwith %}` | зберігає значення у змінну (щоб не рахувати двічі) |
| `{% now "d.m.Y" %}` | поточна дата/час |
| `{% comment %}...{% endcomment %}` | коментар (не потрапляє в HTML) |
| `{% lorem 3 p %}` | заглушка-текст для верстки |
| `{% cycle 'odd' 'even' %}` | чергує значення по ітераціях циклу (смугасті рядки) |
| `{% autoescape off %}...{% endautoescape %}` | вимикає екранування (обережно!) |

```django
{% with price=product.price|floatformat:2 %}
  <span>Ціна: {{ price }} грн</span>       {# порахували один раз, використали кілька #}
{% endwith %}
```

## Фільтри: обробка значень при виводі

> **Фільтр** — це функція, яку застосовують до значення через вертикальну риску `|`, щоб змінити його вигляд **при виводі**, не чіпаючи дані у view.

**Визначення.** Фільтр перетворює значення прямо в шаблоні: форматує дату, обрізає текст, підставляє запасне значення. Аргумент передають через двокрапку: `значення|фільтр:"аргумент"`.

**Як це працює** (приклади з різних доменів):

```django
{{ post.created|date:"d.m.Y" }}          {# 03.07.2026 — блог #}
{{ movie.tagline|default:"—" }}          {# запасне значення, якщо порожньо — кіно #}
{{ products|length }}                     {# кількість елементів — магазин #}
{{ post.body|truncatewords:20 }}          {# перші 20 слів + «…» #}
{{ product.price|floatformat:2 }}         {# 499.00 — рівно 2 знаки #}
{{ order.total|floatformat:0 }}           {# 499 — без копійок #}
{{ movie.is_new|yesno:"Новинка,Класика" }} {# True→Новинка #}
{{ comment.created|timesince }}           {# «3 дні тому» — форум #}
{{ post.body|linebreaks }}                {# переносить \n у <p>/<br> #}
```

Коротка таблиця найчастіших:

| Фільтр | Робить | Приклад результату |
|---|---|---|
| `date:"d.m.Y"` | форматує дату/час | `03.07.2026` |
| `time:"H:i"` | форматує час | `14:30` |
| `timesince` / `timeuntil` | «скільки минуло/лишилось» | `3 дні тому` |
| `default:"—"` | підставляє значення, якщо порожньо/`False` | `—` |
| `default_if_none:"—"` | підставляє лише якщо `None` | `—` |
| `length` | довжина списку чи рядка | `12` |
| `truncatewords:20` / `truncatechars:50` | лишає перші N слів / символів | `Текст обірвано…` |
| `lower` / `upper` / `title` / `capfirst` | регістр | `django` / `DJANGO` / `Django Курс` |
| `floatformat:2` | округлює до N знаків | `499.00` |
| `yesno:"Так,Ні"` | `True`/`False` у слова | `Так` |
| `join:", "` | склеює список у рядок | `дія, драма` |
| `pluralize` | закінчення множини | `коментар{{ n|pluralize:"і,ів" }}` |
| `linebreaks` / `linebreaksbr` | переноси рядків у HTML | абзаци |
| `safe` | не екранувати (довіряєш HTML) | сирий HTML |
| `add:5` | додає число | `15` |

**Навіщо.** Форматування — це справа **відображення**, а не даних. Ціна в БД зберігається як число, а те, показати її як `499` чи `499.00`, вирішує шаблон. Так рівень даних (Model) не знає про HTML — це принцип слабкої зв'язності.

> <i class="bi bi-info-circle"></i> Фільтри можна ланцюжити: `{{ post.body|striptags|truncatewords:10 }}` спершу прибере HTML-теги, потім обріже. Читається зліва направо, як конвеєр.

> <i class="bi bi-exclamation-triangle"></i> Фільтр `safe` вимикає екранування — застосовуй лише до HTML, якому **довіряєш** (не до того, що ввів користувач), інакше відкриваєш XSS.

## Де це в проєкті

Ці теги й фільтри — у **кожному шаблоні**, у будь-якому домені:

- **Список фільмів** (`movie_list.html`): `{% for movie in movies %}{% empty %}` + `{% if movie.rating >= 8 %}` + `{% url 'movie_detail' movie.id %}`.
- **Стаття блогу** (`article_detail.html`): `{{ post.created|date:"d.m.Y" }}`, `{{ post.body|linebreaks }}`, `{{ post.body|truncatewords:40 }}` для анонсу.
- **Кошик магазину** (`cart.html`): `{{ item.price|floatformat:2 }}`, `{% with total=... %}`, `товар{{ n|pluralize:"и,ів" }}`.
- **Форум** (`thread.html`): `{{ comment.created|timesince }}`, `{% for comment in comments %}{% empty %}`.
- **Базовий шаблон** (`base.html`): `{% load static %}`, `{% static 'css/style.css' %}`, `{% block content %}`, форми з `{% csrf_token %}`.

## Паралель із Jinja2 (Flask)

Синтаксис навмисно схожий — `{% ... %}` для логіки, `{{ ... }}` для виводу, `|` для фільтрів. Якщо ти писала шаблони у Flask, тут відчуєш себе вдома. Але це **інший рушій**, і дрібниці відрізняються:

| Аспект | Jinja2 (Flask) | Django Templates |
|---|---|---|
| Умови/цикли | `{% if %}`, `{% for %}` — майже однаково | так само |
| Виклик методів | можна: `{{ user.get_name() }}` | **не можна** дужок: `{{ user.get_name }}` |
| Фільтр аргументи | `truncate(20)` | `truncatewords:20` (двокрапка) |
| URL | `{{ url_for('name') }}` | `{% url 'name' %}` |
| Статика | `{{ url_for('static', ...) }}` | `{% static '...' %}` (з `{% load static %}`) |
| Наслідування | `{% extends %}` + `{% block %}` | так само |
| Філософія | гнучкий, дозволяє логіку | навмисно обмежений |

> <i class="bi bi-exclamation-triangle"></i> Головна пастка при переході: у Django-шаблоні **не можна викликати функції з аргументами** й немає круглих дужок. Django свідомо обмежує логіку в шаблонах — складне має жити у view, а не в HTML. Це прямий наслідок принципу слабкої зв'язності.

## Типові помилки / Нюанси

> <i class="bi bi-exclamation-triangle"></i> **Незакритий блок** (`{% if %}` без `{% endif %}`) → шаблон не скомпілюється. Django не орієнтується на відступи.

> <i class="bi bi-exclamation-triangle"></i> **`{% static %}` без `{% load static %}`** → `Invalid block tag 'static'`. Завантаж бібліотеку тегів угорі шаблону.

> <i class="bi bi-exclamation-triangle"></i> **Дужки у виклику методу** (`{{ obj.method() }}`) → помилка. У Django методи викликаються без дужок.

> <i class="bi bi-exclamation-triangle"></i> **`{% extends %}` не першим рядком** → помилка. `extends` завжди відкриває дочірній шаблон.

## Підсумок

- **`{{ }}`** виводить значення (з автоекрануванням); методи — **без дужок**. **Теги** (`{% ... %}`) керують логікою: `{% if %}/{% elif %}/{% else %}/{% endif %}` і `{% for %}...{% empty %}...{% endfor %}` з лічильником `forloop.counter`. Кожен блок треба закривати.
- **Наслідування**: `{% extends %}` (перший рядок) + `{% block %}` заповнює каркас; `{% include %}` вставляє повторюваний шматок.
- **Теги підключення** працюють з іменами й логічними шляхами: `{% url %}`, `{% static %}` (потрібен `{% load static %}`), обов'язковий `{% csrf_token %}` у POST-формах; є ще `{% with %}`, `{% now %}`, `{% cycle %}`.
- **Фільтри** (`|`) форматують значення при виводі: `date`, `default`, `length`, `truncatewords`, `floatformat`, `yesno`, `timesince`, `pluralize`, `join`, `linebreaks`, `safe`; їх можна ланцюжити.
- Синтаксис близький до **Jinja2**, але це окремий рушій: у Django немає викликів функцій із дужками, а фільтри беруть аргумент через двокрапку. Складну логіку тримай у view, а не в HTML.
