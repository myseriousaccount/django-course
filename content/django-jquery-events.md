# Події DOM: реакція на дії користувача

В уроці «Валідація» ми слухали `blur`/`input`, щоб перевіряти поля. Але події — це взагалі **основа будь-якої інтерактивності**, а не лише валідації: клік відкриває меню, наведення показує підказку, прокрутка ховає хедер. Цей урок — про найуживаніші події для **маніпуляцій зі сторінкою** (на прикладі jQuery, але це універсальні DOM-події).

## Що таке подія

> **Подія** — сигнал, що на сторінці щось сталося: клік, наведення миші, натискання клавіші, прокрутка. Ти «підписуєшся» на подію й виконуєш код у відповідь.

Базовий синтаксис у jQuery:

```js
$('селектор').on('подія', function () {
    // код, що виконається, коли подія станеться
});
```

> <i class="bi bi-info-circle"></i> `.on('click', fn)` — сучасний і рекомендований запис. Старі скорочення (`.click(fn)`, `.hover()`) роблять те саме, але `.on()` універсальніший і підтримує делегування (про нього нижче).

## Об'єкт події (`event`)

Обробник отримує об'єкт події — з нього беруть корисне:

```js
$('#save').on('click', function (e) {
    e.preventDefault();     // скасувати стандартну дію (перехід за <a>, сабміт форми)
    console.log(e.target);  // елемент, на якому сталася подія
});
```

| Що | Навіщо |
|---|---|
| `e.preventDefault()` | скасувати дію за замовчуванням (перехід посилання, відправлення форми) |
| `e.stopPropagation()` | зупинити «спливання» події до батьків |
| `e.target` | елемент, на якому подія виникла |
| `e.key` | яка клавіша натиснута (для клавіатури) |

## Найуживаніші події для маніпуляцій

### Миша: клік, подвійний клік, контекст

```js
// клік — відкрити/закрити меню
$('.menu-toggle').on('click', function () {
    $('#menu').toggle();
});

// подвійний клік — інлайн-редагування
$('.title').on('dblclick', function () {
    $(this).attr('contenteditable', true).focus();
});
```

### Наведення: `mouseenter` / `mouseleave`

Показати підказку чи підсвітити при наведенні й прибрати, коли миша пішла:

```js
$('.card').on('mouseenter', function () {
    $(this).find('.tooltip').fadeIn(150);
}).on('mouseleave', function () {
    $(this).find('.tooltip').fadeOut(150);
});
```

> <i class="bi bi-info-circle"></i> Бери `mouseenter`/`mouseleave`, а не `mouseover`/`mouseout`: перші **не спрацьовують повторно** на дочірніх елементах — саме те, що треба для «навів на картку».

### Клавіатура: `keydown`

Гарячі клавіші, закриття модалки по `Esc`:

```js
$(document).on('keydown', function (e) {
    if (e.key === 'Escape') {
        $('.modal').hide();
    }
});
```

### Прокрутка й розмір вікна: `scroll`, `resize`

```js
// показати кнопку «вгору» після прокрутки на 300px
$(window).on('scroll', function () {
    $('#to-top').toggle($(window).scrollTop() > 300);
});
```

Типові застосування `scroll`: липкий хедер, кнопка «вгору», підвантаження при досягненні низу («load more»), поява елементів під час прокрутки.

### Готовність сторінки

Код, що чіпає елементи, запускають, **коли DOM готовий**:

```js
$(function () {
    // тут усі елементи вже існують — можна вішати обробники
});
```

## Делегування: події для динамічних елементів

Якщо елементи з'являються **пізніше** (додані через AJAX), звичайна прив'язка їх не спіймає — адже на момент `on()` їх ще не було. Рішення — повісити обробник на **батька**, що існує завжди, і вказати селектор дочірніх:

```js
// ❌ не спіймає елементи, додані після завантаження
$('.item').on('click', handler);

// ✅ делегування: слухаємо на контейнері, фільтруємо по .item
$('#list').on('click', '.item', function () {
    $(this).toggleClass('active');
});
```

## Зняти обробник

```js
$('#btn').off('click');   // прибрати всі обробники click з елемента
```

## Одна подія — багато патернів

| Подія | Типове застосування |
|---|---|
| `click` | кнопки, меню, вкладки, лайк, відкрити модалку |
| `dblclick` | інлайн-редагування |
| `mouseenter`/`mouseleave` | тултипи, підсвітка при наведенні |
| `contextmenu` | власне контекстне меню (замість браузерного) |
| `keydown` | гарячі клавіші, `Esc` для закриття |
| `scroll` | липкий хедер, «вгору», lazy-load |
| `resize` | адаптивні підлаштування |
| `submit` | відправлення форми через AJAX (`e.preventDefault()`) |
| `change` | `<select>`, чекбокси, радіо |

## jQuery, vanilla чи Alpine?

Це **ті самі** DOM-події — просто різний запис:

- **jQuery:** `$('#x').on('click', fn)`;
- **чистий JS:** `el.addEventListener('click', fn)` (те саме, трохи довше);
- **Alpine.js:** `@click="..."` прямо в HTML (урок «Alpine.js») — зручно для дрібниць.

jQuery лише скорочує запис і додає делегування; сам набір подій однаковий скрізь.

## Типові помилки / Нюанси

> <i class="bi bi-exclamation-triangle"></i> Прив'язати обробник до елемента, якого **ще немає** (буде доданий пізніше) — не спрацює. Для динамічних елементів використовуй **делегування** (`$('#parent').on('click', '.child', fn)`).

> <i class="bi bi-exclamation-triangle"></i> Забути `e.preventDefault()` на посиланні чи формі — браузер виконає стандартну дію (перехід/перезавантаження), і твій JS «не встигне».

> <i class="bi bi-info-circle"></i> `scroll` і `resize` спрацьовують **дуже часто** (десятки разів на секунду). Для важких дій усередині — застосовуй **throttle/debounce**, інакше сторінка гальмуватиме.

## Підсумок

- **Подія** — сигнал про дію користувача; слухаєш через `$('...').on('подія', fn)` і реагуєш.
- Для маніпуляцій найчастіше: `click`, `mouseenter`/`mouseleave`, `keydown`, `scroll`, `resize`.
- Об'єкт `e` дає `preventDefault()`, `stopPropagation()`, `target`, `key`.
- Для **динамічних** елементів — **делегування** через батька.
- Це універсальні DOM-події: jQuery, чистий JS і Alpine лише по-різному їх записують.
